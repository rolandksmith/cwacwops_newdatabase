  • Students
    • Response
      • blank - No response
      • Y - Wants to take Class
      • R - Refused the Class

    • Student Status
      • C - Student has been replaced. Is not on the waiting list
      • N - Advisor declined replacement student
      • R - Advisor has requested a replacement. Student will not be put on the waiting list
      • S - Assigned advisor has not yet verified student
      • V - Advisor has requested a replacement. Student will be unassigned and put on hold
      • Y - Student verified desire to take the class

    • Intervention Required
      • H - On Hold. See Hold Reason Code for explanation

    • Promotable
      • P - Advisor says student is ready for the next level
      • N - Advisor says student not ready for the next level
      • Q - Advisor quit (so we don't look at two fields)
      • W - Student withdrawn from class

    • Hold Reason Code (for Intervention Required)
      • X - Don’t assign to same Assigned Advisor
      • E - Student not evaluated but signed up for higher level of class. On Hold
      • H - Student not promotable but signed up for next level of class. On Hold
      • Q - Advisor quit, student not evaluated, but signed up for next level of class. On Hold
      • W - Student withdrew from a class, but signed up for the next level of class. On Hold
      • M - Student moved, but signed up for the next level of class. On. Hold
      • B - Student is in the Bad Actor table and needs clearance to be assigned to a class

    • Hold Override (for Intervention Required)
      • blank - Not assigned
      • Y - Release

    • Class Priority
      • blank - Normal assignment
      • 1 or higher - Priority assignment next semester

    • Abandoned
      • blank - normal ... student did not quit the registration process
      • Y - student abandoned the registration process prior to selecting classes
      • N - student updated the registration record and selected classes

    • no_catalog
      • blank - student registered when the complete catalog was available
      • Y - student registered using the abbreviated catalog
      • N - same as blank

    • Flexible
      • Blank or N - Student schedule choices only
      • Y - Student has a flexible schedule

    • Catalog Options
      • If the student registered before the catalog becomes available, the student can select from a list of times and days that the student would be available. Possible entries in this field:
        • MTM (Monday,Thursday Mornings); MTA (Monday,Thursday Afternoons); MTE (Monday,Thursday Evenings)
        • TFM (Tuesday,Friday Mornings); TFA (Tuesday,Friday Afternoons); TFE (Tuesday,Friday Evenings)
        • SWM (Sunday,Wednesday Mornings); SWA (Sunday,Wednesday Afternoons); SEE (Sunday,Wednesday Evenings)
        • STM (Sunday,Thursday Mornings); STA (Sunday,Thursday Afternoons); STE (Sunday,Thursday Evenings)

    • Advisors
      • Survey Score
        • 1 to 5 - (5 being best)
        • 6 - Do not assign to teach - the field carries forward
        • 7 - Assign any previous student to previous advisor
        • 8 - Advisor requires intervention, prior to class assignments
        • 9 - Not completed Evaluation - Can be voided by using CWA webpage sign up
        • 13 - Signed up as an Advisor while trying to take class(es) or not promotable for the teaching level

    • Advisor Response
      • Y - Wants to teach this semester
      • R - Declined teaching this semester

    • Pre-assign Student to an Advisor
      • Make Student Available
        • Student "Verify" must be "Y"
        • "Student Status" must be blank
        • Put student in correct semester

    Usage Details

    Student Response
    If the student registers before students are sent the verification email (appx 45 days before the semester starts), the response field is set to empty. If the student registers after the verification emails are sent, the response is automatically set to Y (student is verified). If the student responds to the verification request that the student does not want the class, response is set to R.
    Waiting List
    If the student registers after students have been assigned to classes, the system attempts to find a suitable class for the student. If the suggestions do not meet the student's needs, the student is placed on a waiting list by setting the waiting list field to N.

    Note that if a suitable class is found and the advisor is accepting students, then the student is automatically assigned to the advisor.
    No Catalog
    If the student registers before the catalog is available, no_catalog is set to Y, indicating that the student did not have access to the class catalog. The student is given a generic list of days and times from which the student can indicate the sutdent's preferences for taking a class. These preferences are stored in the student_catalog_options field. No_catalog is set to N when the student verifies and makes catalog class selections.
